Install Ubuntu Server (24.04 LTS and 26.04 LTS)

Two current Ubuntu Server LTS releases are available:

  • Ubuntu 24.04 LTS (Noble Numbat) — supported until June 2029. Current point release: 24.04.4 with Linux kernel 6.17.
  • Ubuntu 26.04 LTS (Resolute Raccoon) — released April 23, 2026. Supported until 2031. GNOME 50 on desktop, sudo-rs as default sudo provider.

For a new server deployment in 2026, 26.04 LTS is the better choice unless you need 24.04 for compatibility with existing infrastructure.

Download

  • Ubuntu 26.04: ubuntu.com/download/server
  • Ubuntu 24.04: releases.ubuntu.com/24.04

Flash to USB with balenaEtcher or use dd on Linux/macOS:

# Replace /dev/sdX with your USB device
sudo dd if=ubuntu-server-amd64.iso of=/dev/sdX bs=4M status=progress oflag=sync

Boot and Install

Boot from USB. The installer is text-based (Subiquity).

Language: English — better compatibility with documentation and error messages.

Keyboard: Select yours.

Network: Configure here for a static IP (recommended for servers):

Subnet:  192.168.1.0/24
Address: 192.168.1.100    # a free IP on your network
Gateway: 192.168.1.1      # your router
DNS:     1.1.1.1,8.8.8.8

Storage: Use the full disk with LVM unless you have specific needs. LVM lets you resize volumes later without repartitioning.

Profile: Create a username and password. This is your primary login — not root.

OpenSSH server: Check this — it is how you access the server after install.

Snaps: Skip all snap packages here. Install them manually later if needed.

Confirm and install. Reboot when complete.

First Login

SSH in from another machine:

ssh yourusername@192.168.1.100

First Steps After Install

# Update
sudo apt update && sudo apt full-upgrade -y

# Install essential tools
sudo apt install -y curl wget git htop nano ufw fail2ban

# Enable firewall
sudo ufw allow ssh
sudo ufw enable
sudo ufw status

Extend LVM to Use Full Disk

Ubuntu server sometimes does not use the full disk by default:

# Check if there is free space in the volume group
sudo vgs

# Extend and resize in one command
sudo lvextend -l +100%FREE --resizefs /dev/ubuntu-vg/ubuntu-lv

# Verify
df -h

Set Time Zone

sudo timedatectl set-timezone Your/Timezone
timedatectl

Disable Root SSH Login

sudo nano /etc/ssh/sshd_config
# Set: PermitRootLogin no
sudo systemctl restart sshd

Upgrading from 24.04 to 26.04

sudo apt update && sudo apt full-upgrade -y
sudo apt install update-manager-core
sudo do-release-upgrade

For server upgrades, test on a non-production machine first. Ubuntu LTS-to-LTS upgrades are generally reliable but packages occasionally need attention.
